hips.models.Variable¶

class hips.models.Variable(name, id=0, var_type=<VarTypes.CONTINUOUS: 1>, lb=0, ub=None, dim=1)¶

Representation of a variable

This class implements a representation of a variable in the context of linear programming. A variable has an assigned name. A variable has a type, i.e. continuous, integer or binary (see hips.VarTypes). Furthermore, a variable has a lower and upper bound, i.e. the minimum and maximum value a variable may attain. Lastly, each variable has a dimension. If a variable \(x\) has dimension \(n\), then it can be thought of as a column vector containing \(1\)-dimensional variables \(x_j\) with \(j \in \{1, \dots, n\}\).

Note that it is not recommended to explicitly construct a variable via the constructor. Instead use the method hips.models.LPModel.add_variable() provided by hips.models.LPModel.

Example
>>> from hips.models import LPModel
>>> from hips.solver import GurobiSolver
>>> model = LPModel(GurobiSolver())
>>> # Creating variables x and y
>>> x, y = model.add_variable("x", dim=10), model.add_variable("y")
__init__(name, id=0, var_type=<VarTypes.CONTINUOUS: 1>, lb=0, ub=None, dim=1)¶

Constructor

Parameters

  • name – Name of the variable

  • id – Id of the variable. By default the id is 0

  • var_type – Type of the variable, either continuous, integer or binary (see hips.VarTypes)

  • lb – Lower bound of the variable. Has to be of type hips.models.HIPSArray

  • ub – Upper bound of the variable. Has to be of type hips.models.HIPSArray

  • dim – Dimension of the variable. By default the dimension is \(1\)
